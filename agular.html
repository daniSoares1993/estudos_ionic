Explicando o Angular e TypeScript

Framework Angular

  • Junto com o Ionic iremos utilizar o framework angular;
  • Utilizando o ionix start,nos ja criamos uma base de estrutura
  • O Angular é um framework modular que utiliza o javascript;

Estrutura de projeto Angular

  • Estrutura de arquivos com algumas pastas; dist,e2e,node_modules,src e alguns arquivos json;
  • Pasta src => app => Arquivos relacionados ao seu app => home indica a pagina inicial do app;
  • Pasta src => assets => onde conseguimos colocar recursos como imagens...
  • pasta src => environments => onde fazemos as configurações de ambiente
  • pasta src => theme => temos algumas predefiniçoes de scss;
  • src => global.scss aquivos de importação de estilos;
  • src => index.html => nele que ficarao os codigos e onde serao, carregados os componentes do angular dentro da pagina inicial do seu app.

SRC => INDEX.HTML

  • Dentro de index.html,teremos uma estrutura dentro do body, nela teremos uma tag app-root,dentro dessa tag que serao feitos os carregamentos de nossas paginas;
  • main.ts => faz o import de alguns recursos do angular

Módulo oque é?

  • Basicamente modulo é uma maneira de voce organizar suas aplicação em blocos de funcionaliades, e esses blocos de funcionalidades são chamados de componentes
  • Um modulo ele serve para organizar sua aplicação em blocos de funcionalidades;
  • Imagine que voce tenha uma aplicação e dentro dessa aplicação,ela tenha um modulo financeiro;
  • modulo fincandeiro => componentes ; componentes => lista de contas; compontentes => adicionar uma conta; componentes => graficos
  • Ou seja todos esses componentes ficam dentro de uma estrutura maior, dentro de uma aplicação com blocos de finalidades que chamamos de modulos

TypeScript

  • É uma maneira de adicionar ainda mais recursos ao javascript;
  • Dentre as suas caracteristicas ele trabalha a compatibilidade entre os navegadores;

App.componentes.ts/html

    App.componentes.ts

  • Onde temos a inicialização de alguns recursos que iremos utilizar.

    • app.componentes.html

  • temos o ion app /ion router outlet que cria as rotas da nossa aplicação.
  • Basicamente as páginas os caminhos que queremos utilizar para abrir uma determinada pagina;

    • App.routing.modules.ts

  • Define as rotas quando escrevemos em nossa aplicação

Main.ts

Faz o carregamento do nosso modulo inicial,no caso app.module.ts, app.module.ts => inicializa o nosso componente [app.component]; app.component.ts => faz a inicialização dos componentes,e dentro dele é onde iremos carregar o nosso component.html
